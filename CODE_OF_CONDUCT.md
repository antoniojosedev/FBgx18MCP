# Code of Conduct

This is a small open-source project. The rules are short on purpose.

## What's expected

- **Be direct, be technical.** Disagreement on code and design is welcome — that's how the project gets better. Argue the position, not the person.
- **Assume good faith.** Most of the friction in OSS is misread tone, not actual hostility. Re-read before you reply.
- **Speak the language you're comfortable in.** Issues and PRs in English, Portuguese, or Spanish are all fine. Don't gatekeep based on grammar.
- **Stay on topic.** Issues are for things related to this MCP server, GeneXus integration, or its CLI. Not a venue for general GeneXus support, AI-tooling debates, or unrelated complaints.

## What isn't tolerated

- Personal attacks, slurs, harassment, sustained derailing.
- Posting other people's private info (KB contents, credentials, internal company details — yours or theirs).
- "Helpful" PRs that are obvious AI-generated spam: rewriting the README for SEO, adding badges no one asked for, broken auto-translations, etc. These get closed without discussion.

## Enforcement

I'm the only maintainer. If someone crosses a line, I'll:

1. Ask them to stop (once).
2. If it continues, lock the thread.
3. If it keeps continuing, block.

If you think *I'm* the one out of line, email `lucassouza@univali.br` and tell me directly. I'd rather hear it than not.

## Scope

This applies to GitHub issues, pull requests, discussions, and any communication that references this project. What you do elsewhere is your business unless it spills back into the project.
